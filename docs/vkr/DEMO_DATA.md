# Демо-данные OpenPay

Локальная база `OpenPay.Web/openpay-dev.db` подготовлена для демонстрации ВКР. Старый вариант базы сохранен в `OpenPay.Web/DatabaseBackups`.

| Роль | Логин | Пароль |
|---|---|---|
| Администратор платформы | `platformadmin@openpay.local` | `Admin123!` |
| Администратор организации | `admin@openpay.local` | `Admin123!` |

## Организации

- `ООО "Открытые Платежи"` - основная демонстрационная организация.
- `ООО "Бета Трейд"` - вторая организация для проверки изоляции данных.

## Банковские подключения

| Код | Название | Назначение |
|---|---|---|
| `TBANK_SANDBOX` | Основное подключение Т-Банк Sandbox | Проверка реальной песочницы Т-Банка |
| `MOCK` | Основное подключение Mock Bank | Заглушка без внешнего API |

Для `TBANK_SANDBOX` используется тестовый токен `TBankSandboxToken`.

## Маршруты согласования

- `Стандартный маршрут до 100 000` - общий sequential-маршрут.
- `Крупные платежи` - parallel-маршрут для сумм от `100000.01`.
- `Аренда и офисные расходы бухгалтерии` - маршрут по типу расхода `Аренда` и подразделению `Бухгалтерия`.
- `Закупки до 500 000` - маршрут по типу расхода `Поставщики` и подразделению `Закупки`.

## Платежи

| Номер | Статус | Сценарий |
|---|---|---|
| `PAY-DEMO-001` | Черновик | Создание и редактирование платежа |
| `PAY-DEMO-002` | На согласовании | Очередь руководителя |
| `PAY-DEMO-003` | Готов к отправке | Ручной fallback "Подписать и отправить" |
| `PAY-DEMO-004` | Исполнен | Успешная отправка и банковское исполнение |
| `PAY-DEMO-005` | Ошибка | Банковская ошибка |
| `PAY-DEMO-006` | На доработке | Возврат руководителем |
| `PAY-DEMO-007` | На согласовании | Крупный платеж и parallel-маршрут |

## Выписка и аудит

В базе есть демо-выписка по счету T-Банк Sandbox:

- 1 операция сопоставлена с платежом `PAY-DEMO-004`;
- 1 операция оставлена несопоставленной для демонстрации сверки.

В журнале аудита есть события создания платежа, отправки на согласование, утверждения, исполнения банком, банковской ошибки и сверки выписки.

## Повторная подготовка базы

Если нужно заново подготовить базу:

```powershell
dotnet run --project OpenPay.Web -- --seed-demo
```

Если нужен полностью чистый старт, сначала удалите или переименуйте `OpenPay.Web/openpay-dev.db*`, затем выполните команду выше.
