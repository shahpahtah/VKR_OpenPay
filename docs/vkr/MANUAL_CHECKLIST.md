# Ручной чек-лист для проверки OpenPay

Используй этот чек-лист перед демонстрацией проекта на защите ВКР.

## Подготовка

- [ ] Закрыть все запущенные экземпляры `OpenPay.Web` в Visual Studio и терминалах.
- [ ] Выполнить сборку: `dotnet build OpenPay.Web\OpenPay.Web.csproj --no-restore -v:minimal /m:1 /p:UseSharedCompilation=false /p:BuildInParallel=false`.
- [ ] Выполнить тесты: `dotnet test OpenPay.Tests\OpenPay.Tests.csproj --no-restore -v:minimal /m:1 /p:UseSharedCompilation=false /p:BuildInParallel=false`.
- [ ] Запустить web-проект.
- [ ] Открыть главную страницу приложения.

## Тестовые пользователи

- [ ] `accountant@openpay.local` / `Accountant123!` - бухгалтер.
- [ ] `manager@openpay.local` / `Manager123!` - руководитель.
- [ ] `admin@openpay.local` / `Admin123!` - администратор организации.
- [ ] `platformadmin@openpay.local` / `Admin123!` - платформенный администратор.

## Проверка ролей

- [ ] Бухгалтер видит платежи, контрагентов, счета, отчеты, календарь, выписки.
- [ ] Руководитель видит очередь согласований.
- [ ] Руководитель не может открыть платежи, отчеты и календарь бухгалтера.
- [ ] Администратор организации видит пользователей, банковские подключения и маршруты согласования.
- [ ] Платформенный администратор видит управление организациями.

## Платежный сценарий

- [ ] Зайти бухгалтером.
- [ ] Создать контрагента или выбрать существующего.
- [ ] Проверить, что у организации есть счет с банковским подключением.
- [ ] Создать платеж с уникальной суммой, чтобы не сработала проверка дублей.
- [ ] Убедиться, что платеж создан в статусе `Черновик`.
- [ ] Отправить платеж на согласование.
- [ ] Зайти руководителем.
- [ ] Открыть очередь согласований.
- [ ] Убедиться, что платеж отображается в очереди.
- [ ] Открыть платеж и утвердить.
- [ ] Зайти бухгалтером или администратором.
- [ ] Проверить, что платеж получил демо-подпись.
- [ ] Проверить итоговый статус `Отправлен в банк` или `Готов к отправке` с понятным сообщением банка.

## Негативные проверки платежей

- [ ] Создать второй платеж с тем же контрагентом, датой и суммой.
- [ ] Убедиться, что система показывает ошибку возможного дубля.
- [ ] Создать платеж с валютой, отличной от валюты счета.
- [ ] Убедиться, что система не дает сохранить некорректный платеж.
- [ ] Руководителем отклонить платеж без комментария.
- [ ] Убедиться, что система просит комментарий.
- [ ] Руководителем вернуть платеж на доработку с комментарием.
- [ ] Убедиться, что бухгалтер видит статус `На доработке`.

## Банковские подключения

- [ ] Зайти администратором.
- [ ] Открыть `Банковские подключения`.
- [ ] Создать подключение `SBER` или `TBANK` с демо-токеном.
- [ ] Проверить, что токен в списке не отображается открытым текстом.
- [ ] Создать или отредактировать счет организации и выбрать созданное подключение.
- [ ] Создать платеж по этому счету и провести его через согласование.
- [ ] Проверить, что отправка идет через выбранный адаптер.

## T-Банк Sandbox

- [ ] Создать подключение с банком `TBANK_SANDBOX`.
- [ ] В access token указать `TBankSandboxToken`.
- [ ] Создать счет, привязанный к этому подключению.
- [ ] Создать и утвердить тестовый платеж.
- [ ] Если возникает TLS/сертификатная ошибка, проверить запуск вне sandbox-среды Codex и доступ к `https://business.tbank.ru/openapi/sandbox`.

## Выписки и сверка

- [ ] Открыть `Выписки`.
- [ ] Выбрать счет, у которого настроено банковское подключение.
- [ ] Выбрать период.
- [ ] Нажать `Загрузить и сверить`.
- [ ] Проверить количество операций, matched и unmatched.
- [ ] Нажать повторную сверку у загруженной выписки.
- [ ] Выбрать счет без подключения.
- [ ] Убедиться, что система показывает понятную ошибку.

## Маршруты согласования

- [ ] Открыть `Маршруты согласования`.
- [ ] Создать маршрут с диапазоном сумм.
- [ ] Создать маршрут с типом расхода.
- [ ] Создать маршрут с подразделением счета.
- [ ] Проверить, что маршрут сохраняется.
- [ ] Проверить ошибку, когда минимальная сумма больше максимальной.
- [ ] Отключить маршрут и убедиться, что он больше не активен.

## Отчеты и экспорт

- [ ] Открыть `Отчеты`.
- [ ] Проверить сводку по статусам.
- [ ] Проверить группировку по контрагентам.
- [ ] Выбрать период фильтра.
- [ ] Скачать CSV.
- [ ] Скачать Excel.
- [ ] Открыть файлы и проверить наличие строк платежей.

## CSV-импорт

- [ ] Импортировать корректный CSV контрагентов.
- [ ] Импортировать CSV контрагентов с ошибочным ИНН.
- [ ] Проверить, что ошибочные строки не скрываются.
- [ ] Импортировать корректный CSV платежей.
- [ ] Проверить, что платежи созданы как черновики.

## Аудит и UI

- [ ] Открыть главную страницу админки.
- [ ] Проверить последние события аудита.
- [ ] Убедиться, что события платежей, подключений, маршрутов и выписок попадают в журнал.
- [ ] Просмотреть ключевые страницы на предмет mojibake-текста.
- [ ] Проверить формы на ширине мобильного экрана.
- [ ] Проверить пустые состояния в разделах без данных.
