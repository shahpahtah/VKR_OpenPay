# Тест-кейсы OpenPay для ВКР

Документ фиксирует набор тест-кейсов, который покрывает требования ВКР: платежи, согласование, банковские подключения, выписки, сверку, отчеты, аудит, роли и базовую безопасность.

## Условные обозначения

- `P0` - критичный сценарий, без него прототип нельзя демонстрировать.
- `P1` - важный сценарий для защиты ВКР.
- `P2` - дополнительная проверка качества.
- `Auto` - проверяется автотестом или скриптом.
- `Manual` - проверяется вручную через UI.
